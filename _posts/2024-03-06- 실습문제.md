문제5
app crashed 발생
app crashed 에러는 원인은 다양하다.

"[nodemon] app crashed - waiting for file changes before starting..." 메시지는 nodemon이 실행 중인 앱이 충돌로 인해 비정상 종료되었음을 나타낸다. 이 메시지는 파일 변경을 감지하고 앱을 다시 시작하기 전에 nodemon이 기다리고 있다는 것을 의미한다.

앱이 충돌로 인해 종료되는 이유는 다양할 수 있다. 주요 원인 중 하나는 코드에 오류가 있는 경우인데 제일 먼저 오타를 확인해보자. 오타가 없다면 아래 순서대로 확인해본다.

터미널 창에서 오류 메시지 확인: 앱이 충돌하면 일반적으로 터미널 창에 오류 메시지가 표시된다. 오류 메시지를 찾아서 어떤 문제가 있는지 확인해보자.

코드 검토: 앱이 충돌하는 이유를 파악하기 위해 코드를 다시 확인해본다. 문법, 잘못된 변수 사용, 함수 호출 오류 등과 같은 일반적인 실수를 확인하면서 오류를 찾아본다.

디버깅: 앱이 충돌하는 특정 부분을 찾기 위해 디버깅을 해보자. Node.js 디버거를 활용하거나 console.log()를 사용하여 코드 실행 중에 중간 결과를 확인할 수 있다. 디버깅을 통해 코드의 문제를 식별하고 해결할 수 있다.

dependencies 부분 확인: 종종 충돌은 앱의 dependencies 문제로 인해 발생한다. package.json 파일을 확인하여 dependencies에 충돌이 있는지 확인하고 필요한 dependencies을 업데이트해보자. npm update또는 yarn upgrade을 사용하여 종속성을 업데이트할 수 있다.

파일위치 다시 확인해보기: 파일 경로가 잘못돼어 위치를 바꾸니까 성공적으로 실행됐다.

시스템 재부팅: 시스템 자원 또는 다른 프로세스와의 충돌로 인해 앱이 충돌할 수 있다. 컴퓨터를 재부팅하고 다시 시도해 보자.

나는 패키지 모듈을 다운로드 받은 거이기에 코드 검토 디버깅 필요하기에 하지 않다

해결
아예 다시 설치하고 dev를 삭제하지 않고, auidt fix와 audit fix --force로 몇 개의 보안문제를 해결해주었고, 1개의 치명적인 보안문제는 넘어가서 실행해보니 노드가 정상적으로 구동됐다.
dev도 중요한 것이니 함부로 삭제하지 않는게 좋을 것 같고, 현재 내게 중요한 것은 보안보다 실습하는 것이었기에 그냥 지나가 주었다
1개의 치명적 보안문제는 버전이 달라서 문제가 되었는데 아마 해당 원체인이 4년전 것이다 보니 audit fix를 해도 통하지 않은 것 같다
