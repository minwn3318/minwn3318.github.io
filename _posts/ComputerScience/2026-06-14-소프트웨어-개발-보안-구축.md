# 정보처리기사 - 소프트웨어 개발 보안 구축   
   
### 1. Secure SDLC   
   
#### - 소프트웨어 개발 보안 요소 
```
: 소프트웨어 개발에 있어 충족시켜야할 요소 및 요건을 의미한다   
```   
   
이 보안 요소 중 가장 중요한 요소를 **보안의 3대 요소** 라 하며, 각각 기밀성, 무결성, 가용성을 의미한다.
   
- 기밀성 (confidentiality)   
```
: 시스템 내의 정보와 자원은 인가된 사용자 외에는 접근이 불가한 성질을 의미한다.
```
정보가 전송 중에 노출되더라도 데이터를 읽을 수 없다.   
   
- 무결성 (Integrity)
```
: 시스템 내의 정보와 자원의 수정은 오직 인가된 사용자만 할 수 있으며, 사용자의 수정후 해당 정보와 자원의 상태는 항상 유지되는 성질을 의미한다
```
   
- 가용성 (Availablilty)
```
: 인가된 사용자는 시스템 내의 정보와 자원을 언제라도 사용할 수 있는 성질을 의미한다.   
```   
  
보안의 3대 요소 외에도 소프트웨어 개발에서 충족되어야 할 요소는 **인증**과 **부인 방지**가 있다.   
   
- 인증 (Authentication)
```
: 시스템 내의 정보와 자원을 사용하려는 사용자가 합법적 사용자인지 확인하는 모든 행위이다
```
대표적인 방법으로 **패스워드**, **인증용 카드**, **지문 검사** 등이 있다   

- 부인 방지 (NonRepudiation)
```
: 데이터를 송수신한 자가 송수신 사실을 부인할 수 없도록, 송수신 증거를 제공한다.
```

### 2. 입력 데이터 검증 및 표현      
```
: 보안문제가 구현 단계의 입력 데이터에서부터 발생하기에 해당 단계에서 검증, 점검하는 내역들을 의미한다 
``` 
아래에는 보안 점검 항목들이 나열되어 있다.   

#### - SQL 삽입   
```
: 웹 응용 프로그램에 SQL을 삽입하여, 내부 데이터베이스 서버의 데이터를 유출 및 변조하고, 관리자 인증을 우회하는 보안 약점이다
```   
동적 쿼리에 사용되는 입력 데이터에 예약어 및 특수문자가 입력되지 않게 필터링 되도록 설정하여 방지할 수 있다.   

### 3. 코드 오류   
```
: 소프트웨어 응용 단계에서 개발자들이 코딩 중 실수하기 쉬운 오류들을 예방하기 위한 내역들을 의미한다
```   
아래에는 예방 내역들이 나열되어 있다.   

#### - 스택 가드   
```
: 주소가 저장되는 스택에서 발생하는 보안 약점을 막는 기술을 의미한다.
```
메모리상에서 프로그램의 복귀 주소와 변수 사이에 특정 값을 저장한 후, 그 값이 변경되었을 경우 **오버플로우** 상태로 판단하여 프로그램을 중단시킨다.   

### 4. 캡슐화   
```
: 정보 은닉이 필요한 데이터와 기능을 불완전하게 캡슐화하여 잘못 사용함으로써 발생할 수 있는 문제를 예방하기 위한 보안 점검 항목들을 의미한다
```   

캡슐화의 보안 약점을 예방하기 위해서는 먼저 캡슐화의 핵심은 접근 제어자에 대해 잘 숙지해두어야한다.   

#### - 접근 제어자   
```
: 프로그래밍 언어에서 특정 개체를 선언할 때 외부로부터의 접근을 제한하기 위한 예약어를 의미한다
```

접근제어자들은 다음 표와 같이 접근이 가능하다.   

| 접근제어자  | 클래스 내부 | 패키지 내부 | 하위 클래스 | 패키지 외부|
| --------- | ---------- | --------- | ---------- | -------- |
| Public    |      O     |     O     |     O      |     O    |
| Protected |      O     |     O     |     O      |     X    |
| Default   |      O     |     O     |     X      |     x    |
| Private   |      O     |     X     |     X      |     x    |

### 5. 암호 알고리즘   
```
: 중요 정보를 보호하기 위해 평문을 암호화된 문장으로 만드는 절차 또는 방법을 의미한다
```   
단순히 비밀유지를 위해 사용하기도 하지만, 정보화 사회에서는 **보안의 3대 요소**를 지키기 위해 적극적으로 사용된다.   
   
암호 방식 분류로는 다음과 같다   
   
- 암호화 방식

- - 양방향
- - - 개인키
- - - - 스트림 방식
- - - - 블록 방식

- - - 공개키

- - 단반향
- - - 해시

이어서 각 암호화 방식에 대해 설명하도록 한다.   

#### - 개인키 암호화 기법   
```
: 동일한 키로 데이터를 암호화하고 복호화하는 암호화 기법을 의미한다
```   
개인키 암호화 기법은 **대칭 암호 기법** 또는 **단일키 암호화 기법**이라고도 한다. 암호화/복호화 속도가 빠르다는 장점이 있으나, 사용되는 대상이 많아질수록 관리해야 할 키의 수가 많아진다는 단점이 존재한다.   
  
방식에는 **스트림 암호화 방식**과 **블록 암호화 방식**이 존재한다

- 스트림 암호화 방식
```
: 평문과 동일한 길이의 스트림을 생성하여 비트 단위로 암호화하는 방식을 가리킨다
```
종류로 LFSR, RC4, TKIP가 존재한다.   

- 블록 암호화 방식
```
: 한 번에 하나의 데이터 블록을 암호화 하는 방식을 가리킨다
```
종류로 DES, SEED, AES, ARIA, IDEA, Skipjack

#### - 공개키 암호화 기법   
```
: 암호화 할 때 사용하는 공개키를 사용자에게 공개하고, 복호화 할 때의 비밀키를 관리자가 비밀리에 관리하는 암호화 기법을 의미한다
```
공개키 암호화 기법은 **비대칭 암호 기법**이라고도 한다. 사용되는 대상이 많아져도 관리해야할 키의 수가 적다즌 장점이 있으나, 암호화/복호화 속도가 느리다는 단점이 존재한다   

대표적으로 RSA와 ECC기법이 있다.

#### - 알고리즘의 종류   

- IDEA
```
: 스위스의 라이와 메시가 1990년에 개발한 PES를 개선한 알고리즘이다
```
블록 크기는 64비트이고, 키 길이는 128비트이다.

- SkipJack
```
: 국가 안전 보장국 (NSA)에서 개발한 암호화 알고리즘이다
```
**클리퍼 칩**이라는 IC 칩에 내장되어 있다.

블록 크기는 64비트이고, 키 길이는 80비트이다.

주로 **음성 통신 장비**에 삽입되어, 음성 데이터를 암호화한다.

- DES
```
: 1975년 미국 NBS에서 발표한 개인키 암호화 알고리즘이다
```

블록크기는 64비트, 키 길이는 56비트이며 16회의 라운드를 수행한다

- AES
```
: 2001년 미국 표준 기술 연구소 (NIST)에서 발표한 개인키 암호화 알고리즘이다
```

DES의 한계를 느낀 NIST에서 공모한 후 발표하였다.

블록 크기는 128비트이며, 키 길이에 따라 AES-128, AES-192, AES-256으로 분류된다

- RSA
```
: 1978년 MIT의 라이베스트, 샤미르, 애들먼에 의해 제안된 공개키 암호화 알고리즘이다
```

큰 숫자를 소인수분해 하기 어렵다는 것에 기반하여 만들어졌다.

- TKIP
```
: 기존의 무선 랜 보안 프로토콜인 WEP의 취약성을 보완한 데이터 보안 프로토콜이다
```

암호 알고리즘의 입력 키 길이를 128비트로 늘리고 패킷당 키 할당, 키값 재설정 등의 키 관리 방식을 개선하였다.

#### - 해시   
```
: 입력 데이터나 메시지를 고정된 길이의 값이나 키로 변환하는 것을 의미한다
```

**해시 알고리즘**을 **해시 함수**라고 부르며, 해시 함수로 변환된 값이나 키를 **해시값** 또는 **해시키**라고 부른다.

종류로 SHA 시리즈, HAVAL, MD4, MD5, N-NASH, SNEFRU가 있다

- MD4
```
: 1991년 R Rivest가 MD4를 대처하기 위해 고안한 암호화 해시 함수이다
```

블록크기가 512비트이며, 키 길이는 128비트이다.

### 6. 서비스 공격 유형   
```
: 표적이 되는 서버의 자원을 고갈시킬 목적으로 다수의 공격자 또는 시스템에서 대량의 데이터를 한 곳으로 서버에 집중적으로 전송함으로써 표적이 되는 서버의 정상적인 기능을 방해하는 것을 가리킨다
```

아래는 서비스 공격 유형 모음들이다   

- Ping of Death   
```
: 패킷 크기를 허용범위 이상으로 전송하여 네트워크를 마비시키는 서비스 거부 공격 방법이다 
```
   
- SMURFING   
```
: IP나 ICMP의 특성을 악용하여 엄청난 양의 데이터를 한 사이트에 집중적으로 보냄으로써 네트워크를 불능상태로 만드는 공격 방법
```

- LAND Attack   
```
: 패킷을 전송할 때 송신 IP주소와 수신 IP 주소를 모두 공격 대상의 IP주소로 하여 자신에 대해 무한히 응답하게 하는 공격 방법이다
```   

이후로 네트워크와 보안 침해 공격 유형 모음에 대해 알아보자   

#### - 네트워크 침해 공격 유형
   
- Session Hijacking   
```
: 상호 인증 과정을 거친 후 접속해 있는 서버와 서로 접속되어 클라이언트 사이의 세션 정보를 가로채는 공격기법
```

- ARP Spoofing   
```
: 자신의 물리적 주소(MAC)를 공격 대상의 것으로 변조하여 데이터 패킷을 가로채거나 방해하는 기법
```

- Social Enginnering   
```
: 인간 상호 작용의 깊은 신뢰를 바탕으로 시스템 침입하는 수단
```  

- Dark Data   
```
: 활용되지 않고 저장만 되어 있는 대량의 데이터
```

- Typosquatting   
```
: 주소를 잘못 입력하거나 실수를 이용하기 위해 이와 유사한유명 도메인을 미리 등록하는 것
```
   
- Sniffing      
```
: 네트워크의 중간에서 남의 패킷 정보를 도청하는 유형
```

- Smishing   
```
: 문자 메시지를 이용해서 사용자의 개인 신용 정보를 빼내는 수법
```

- Qshing   
```
: 큐알 코드를 통해 악성 앱의 다운로드를 유도하거나 악성 프로그램을 설치하도록 하는 금융 사기 기번의 하나
```
   
- Spear Phishing  
```
: 사회 공학의 한 기법으로 특정 대상을 선정한 후 그 대상에게 일밙거인 이메일로 위장한 메일을 지속적으로 발송하여 첨부된 파일이나 링크를 클릭하도록 유도하는 공격
```

- APT (advanced Persistent Threats)   
```
: 조직적으로 특정 기업이나 조직 네트워크에 침투해, 활동 거점을 마련한 뒤 때를 기다려 보안을 무력화 시키고 정보를 수집한 다음 외부로 빼돌리는 형태의 공격
```    
- Brute Force Attack   
```
: 암호화된 문서의 암호키를 찾아내기 위해 적용 가능한 모든 값을 대입하여 공격하는 방식
```   

#### - 보안 침해 공격 유형   
   
- Watering Hole   
```
: 웹 사이트를 사전에 감염시켜 대상이 해당 사이트에 방문했을 때 악성 코드에 감염되게 하는 공격
```   

- Key Logger Attack   
```
: 컴퓨터 사용장 키보드 움직임르 타지해 정보를 몰래 빼가는 해킹공격
```   

- Ransomware   
```
: 내부 문서나 파일 등을 암호화해 사용자가 열지 못하게 하는 프로그램
```   

- Back Door   
```
: 시스템 설계자가 액세스의 편의를 위해 시스템 보안을 제거하여 만들어놓은 비밀통로
```

- RootKit   
```
: 불법적인 해킹에 사용되는 기능들을 제공하는 프로그램들의 모음
```
   
- Zombie PC   
```
: 악성 ㅗ드에 감염되어 다른 프로그램이나 컴퓨터를 조종하도록 만들어진 컴퓨터로 C&C 서버의 제어를 받는다
```
주로 디도스 공격 등에 이용된다   

- C&C 서버   
```
: 감염된 좀비 pc에 명령을 내리고 악성코드를 제어하기 위한 용도로 사용되는 서버
```

- Botnet   
```
:  악성 프로그램에 감염되어 악의적인 의도로 사용될 수 있는 다수의 컴퓨터들이 네트워크로 연결된 형태
```

- Zero Day Attack   
```
: 보안 취약점이 발견되었을 때, 널리 공표하기도 전에 보안 공격을 당하는 것
```
공격의 신속성을 의미한다   

- Trojan Horse   
```
: 정상적인 프로그램으로 위장하여, 다른 프로그램이 동작할 때 활성화되지만, 복제능력 없음 
```

- Virus   
```
: 컴퓨터의 정상적인 작동을 방해하기 위해, 운영체제나 저장된 데이터에 손상을 입히는 프로그램으로 복제가 가능하며 감염시킬 수 있다
```

- Worm   
```
: 네트워크를 통해 자식을 복제하여 시스템의 부하를 높여 결국 시스템을 다운시키는 바이러스
```

부산 서비스 거부 공격, 버퍼 오버플로 공격, 슬래머 등이 웜 공격의 한 형태이다   

- Supply Chain Attack   
```
: 소프트웨어 공급망에 침투하여 악성 코드를 배포하는 공격
```   
- Data Diddling   
```
: 처리할 자료를 다른 자료와 바꿔서 처리해, 입력갑이나 출력값을 부정한 의도로 수정하여 잘못된 결과가 나오도록 유도하는 방식
```
   
### 7. 서버 인증   

여기서 말하는 서버와 인증이 무엇인지부터 살펴보자

#### 인증
```
: 컴퓨터 시스템이나 네트워크 시스템에서 로그인을 요청한 사용자의 정보를 확ㅇㄴ하고 접근 권한을 검증하는 보안 절차이다
```

서버의 경우 단순 인터넷을 통한 송수신 시스템을 갖춘 것이 아닌 보안기능을 갖춘 서버를 가리킨다.   

#### - 보안 서버
```
: 인터넷을 통해 개인 정보를 암호화하여 송/수신할 수 있는 기능을 갖춘 서버를 말한다
```
   
앞서 말한 인증에는 여러 종류가 존재한다   
   
- 지식기반인증 (Something You Know)   
> 고정된 패스워드, 아이핀

- 소유기반인증 (Something You Have)   
> 신분증, OTP
   
- 생체기반인증 (Something You Are)   
> 지문, 홍채

- 위치기반인증 (Somewhere You Are)  
> GPS, 콜백, IP

- 행위기반인증 (Something You Do)
> 서명, 동작

그리고 인증과 과련된 서비스 규격 용어로 **AAA**가 있다   
   
- AAA   
```
: 사용자의 컴퓨터 자원 접근에 대한 처리와 서비스를 제공하는 기반 구조 또는 규격을 의미한다   
```
   
> Authentication   
```
: 접근하는 사용자의 신원을 검증하는 기능 
```   

> Authorization   
```
: 신우너이 검증된 사용자에게 특정된 권한과 서비스를 허용하는 기능
```

> Accounting   
```
: 얼마만큼의 자우너을 사용했는지 어떤 종류이 서비스를 이용했는지 기록 및 보관하는 기능
```

### 8. 보안 솔루션
```
: 접근 통제, 침입 차단 및 탐지 등을 수행하여 외부로부터의 불버적인 침입을 막는 기술 및 시스템을 말한다
```
   
아래는 보안 솔루션 모음집이다   
   
- IDS(Intrusiong Detection System)   
```
: 컴퓨터 시스템의 비정상적인 사용, 오용, 남용들을 실시간으로 탐지하는 시스템
```   
**오용 탐지**는 미리 입력해둔 공격 패턴 감지하면 알리고,   
**이상 탐지**는 평균적인 시스템의 상태를 기준으로 비정상적인 행위나 자원의 사용이 감지되면 알려준다   

- VPN   
```
: 공중 네트워크와 암호화 기술을 이요하여 사용자가 마치 자신의 전용 회선을 사용하는것처럼 해주는 보안 솔루션
```   
**SSL VPN**는 ssl 프로토콜을 이용한 vpn이고,   
**IPSec VPN**은 네트워크를 서로 연결하는 방식으로 암ㅎ화를 위해 IPSec프로토콜을 사용한 것을 가리킨다   

- SIEM (Securiy Information and Event Management)   
```
: 로그 및 보안 이벤트를 통합하여 괄니하는 빅데이터 기반의 보안 솔루션이다
```   
- SSH   
```
: 다른 컴퓨터에 원격으로 접소가여 작업을 수행할 수 있도록 다양한 기능을 지원하는 프로토콜을 가리킨다
```

- ISMS (Information Security Managment System)   
```
: 정보자산을 안전하게 보호하기 위한 보호 절차와 대책을 의미한다
```

- BitLocker
```
: 윈도우 전용으 볼륨 암호화 기능   
```  
TPM과 AES-128알고리즘을 사용한다

- ISAC (Information Sharing & Analysis Center)   
```
: 취약점 및 침해요인과 그 대응반안에 관한 정보를 제공하며, 침해 사고가 발생하는 경우 실시간 경보 분석체계를 운영하고 정보통신기반시설을 보호하기 위한 업무를 수행하는 곳이다
```   
- BCP (Business Continuity Planning)
```
: 위기관리를 기반으로 재해복구, 업무복구 및 재개, 비상계획 등의 비즈니스 연속성을 보장하기 위한 계획이다
```
