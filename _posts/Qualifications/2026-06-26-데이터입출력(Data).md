# 정보처리기사 - 데이터 입출력 구현

해당 포스트에서는 정보처리기사 개념문제집 2장 **데이터 입출력 구현**을 정리한다

## 스택
```
: 리스트의 한쪽 끝으로만 자료의 삽입, 삭제 작어빙 이루어지는 자료구조이다
```
- 후입선출 방식으로 자료를 처리한다
- 저장공간이 없는데 젖아하면 오버플로, 삭제할 내용이 없는데 삭제하면 언더플로가 발생한다

---

## 트리 관련 용어
- 노드
> 트리의 기본 요소로서 자료 항목과 다른 항목에 대한 가지를 합친 것
- 근노드
> 트리의 맨 위에 있는 노드
- 디그리
> 각 노드에서 뻗어나온 가지의 수
- 단말노드
> 자식이 하나도 없는 노드
- 레벨
> 근 노드를 기준으로 1이라 하고, 아래 계속이어지는 자식 노드 층
- 깊이
> 최대 레벨
- 숲
> 여러 개의 트리가 보여있는 것
- 트리의 디그리
> 노드들의 디그리 중에서 가장 많은 수

---

## Preorder 운행법
- 루트 -> 왼쪽 -> 오른족 순으로 운행하며 노드를 찾아감

### Inorder 운행법
- 왼쪽 -> 루트 -> 오른쪽 순으로 운행하며 노드를 찾아감

### Postorder 운행법
- 왼쪽 -> 오른쪽 -> 루트 순으로 운행하며 노드를 찾아감

---

## 삽입 정렬
- 이미 순서화된 파일에 새로운 하나의 레코드를 순서에 맞게 삽입시켜 정렬하는 방식

### 선택 정렬
- 최소값을 찾아 첫번째 레코드 위체 놓고 나머지 중에서 다시 최소값을 찾아 두 번째 레코드 위치에 놓는 방식을 반복하여 정렬하는 방식

### 버블정렬
- 인접한 두 개의 레코드 키 값을 비교하여 그 크기에 따라 케로크 위치를 서로 교환하는 정렬 방식

### 퀵 정렬
- 키를 기준으로 작은 값은 왼쪽, 큰 값은 오른쪽 서브 파링레 분해시키는 과저을 반복하는 정렬 방식
