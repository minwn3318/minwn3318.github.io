# 정보처리기사 - 응용 SW 기초 기술 활용   
   
해당 포스트에서는 정보처리기사 개념문제집의 11장 **응용 SW 기초 기술 활용**을 정리한다   

## 1. 운영체제 (Operating System)   
```
: 컴퓨터 시스템의 자원들을 효율적으로 관리하며, 사용자가 컴퓨터를 편리하고 표과적으로 사용할 수 있도록 환경을 젝오하는 여러 프로그램의 모임이다
```   
컴퓨터 사용자와 컴퓨터 하드웨어 간의 인터페이스로 동작하는 시스템 소프트웨어의 일종이다. 운영체제는 프로세서, 기억장치, 입출력 장치, 파일 및 정보등의 자원을 관리한다   

컴퓨터가 발전하면서 많은 기능을 처리하게 되었고, 이것들을 효율적이고 신속하게 처리하기 위한 이제 컴퓨터의 비서와 같은 존재다. 첫번째 챕터에서는 이 운영체제에 대해 알아보자   

### 운영체제 종류   

   운영체제는 하나의 형태만 가지지 않고, 하드웨어 종류에 따라 대표적으로 사용되는 형태들이 존재한다   

   1. UNIX
   ```
   : 1960suseo AT&T 벨 연구소, MIT, General Electric이 공동 개발한 운영체제이다.
   ```   
   시분할 시스템을 위해 설계된 대화식 운영체제이며, C언어로 작성되어 이식성이 높고, 장치와 프로세스 간의 호환성이 높은 특성을 가지고 있다.   

   트리 구조의 파일 시스템을 갖는다.

   윈도우, 맥, 리눅스와 같은 우리주변의 pc 운영체제들은 이 유닉스를 기반으로 만들어졌다.

   2. Android
   ```
   : 구글 사에서 개발한 리눅스 커널 기반의 개방형 모바일 운영체제이다
   ```   
   오픈 소스이며, 자바와 코틀린으로 애플리케이션을 작성한다.

   주로 모바일, 태블릿용 운영체제로 사용된다.

### 기억장치 관리    
   
   수 많은 프로그램들이 기억장치에 적재되고 내려지기 때문에, os는 이 기억장치들을 관리해야한다. 기억장치 관리 전략으로 대표적으로 배치전략이 있다.

   #### 기억장치 관리 전략
   - 배치 전략
   ```
   : 새로 반입되는 프로그램이나 데이터를 주기억장치의 어디에 위치시킬 것인지를 결정하는 전략이다
   ```
   해당 전략에는 3가지의 방법이 있는데, **최초**, **최적**, **최악**이 있다. 
   1. 최초 적합
   ```
   : 첫번째 분할 영역에 배치시키는 방법
   ```
   2. 최적 적합
   ```
   : 단편화를 가장 작게 남기는 분할 영역에 배치시키는 방법
   ```
   3. 최악 적합
   ```
   : 단편화를 가장 많이 남기는 분할 영역에 배치시키는 방법
   ```

   #### 기상기억장치 구현 기법
   - 페이징 기법
   ```
   : 프로그램과 주기억 장치의 영역을 동일한 크기로 나눈 후 나눠진 프로그램을 동일하게 나눠진 주기억장치의 영역에 적재시켜 실행하는 기법이다.
   ```
   > 페이지
   ```
   : 프로그램을 일정한 크기로 나눈 단위를 가리킨다
   ```
   > 페이지 프레임
   ```
   : 페이지 크기롤 일정하게 나누어진 주기억장치의 단위를 가리킨다
   ```

   - 세그먼테이션 기법   
   ```
   : 프로그램을 다양한 크기의 논리적인 단위로 나눈 후 주기억자치에 적재시켜 실행시키는 기법이다
   ```
   > 세그먼트
   ```
   : 논리적인 크기로 나눈 단위를 가리키며, 고유한 이름과 크기를 갖는다
   ```   

   #### Locality
   ```
   : 프로세스가 실행되는 동안 주기억장치를 참조할 때 일부 페이지만 집중적으로 참조하는 성질을 가리킨다
   ```
   스래싱을 방지한 위한 워킹셋 이론의 기반이 되었다.
   - 시간 구역성
   ```
   : 프로세스가 실행되면서 하나의 페이지를 일정 시간 동안 집중적ㅇ로 액세스하는 현상
   ```

   - 공간 구역성
   ```
   : 프로세스 실행 시 일정 위치의 페이지를 집중적으로 액세스하는 현상
   ```

   #### 워킹 셋
   ```
   : 프로세스가 일정 시간 동안 자주 참조하는 페이지들의 집합을 가리킨다
   ```

   #### 스래싱
   ```
   : 프로세스의 처리시간볻 페이지 교체에 소요되는 시간이 더 많아지는 현상이다
   ```

### 프로세스 
```
: 실행중인 프로그램을 의미한다
```  
   
   #### 프로세스 상태 전이 관련 용어
   ```
   : 프로세스가 시스템 내에 존재하는 동안 프로셋의 상태가 변하는 것을 의미하며 이와 관련된 용어를 정리한다
   ```

   - Dispatch
   ```
   : 준비 상태에서 대기하고 있는 프로세스 중 하나가 프로세서를 할당 받아 시행상태로 전이되는 과정이다
   ```

   - Wake Up
   ```
   : 입출력 작업이 완료되어 프로세스가 대기 상태에서 준비상탤 전이되는 과정이다
   ```

   - Spooling
   ```
   : 느린 입출력장치의 처리속도를 보완하고, 다중 프로그래밍 시스템의 성능르 향상시키기 위해 입출력 데이터를 직접 입출력장체 보내지 않고 나중에 한꺼번에 입출력하기 위해 디스크에 저장하는 과정이다
   ```

### 환경변수   
```
: 시스템 소프트웨어의 동작에 영향을 미치는 동적인 값들의 모임
```

### 교착상태
```
: 둘이상의 프로세슫르이 자원을 점유한 상태에서 서로 다른 프로세스가 점유하고 있는 자원을 요구하며 무한정 기다리는 현상을 의미한다
```
   
   #### 교착상태 발생의 필요 충분 조건
   - 상호배제(Mutual Exclusion)
   ```
   : 한 번에 한 개의 프로세스만이 공유 자원을 사용할 수 있어야 함
   ```

   - 점유와 대기(Hold and Wait)
   ```
   : 최소한 하나의 자원을 점유하고 있으면서 다른 프로세스에 할당되어 사용되고 있는 자원을 추가로 점유하기 위해 대기하는 프로세스가 있어야 함
   ```

   - 비선점(Non-preemption)
   ```
   : 다른 프로세스에 할당된 자원은 사용이 끝날 때까지 강제로 빼앗을 수 없어야 함
   ```

   - 환형대기(Circular Wait)
   ```
   : 프로세스들이 원형으로 구성되어 있어 자신에게 할당된 자원을 점유하면서 앞이나 뒤에 있는 프로세스의 자원을 요구해야함
   ```

   #### 교착상태 해결방법
   - 예방 기법(Prevention)
   ```
   : 교착상태가 발생하지 않도로 사전에 시스템을 제어하는 방법
   ```

   - 회피 기법(Avoidance)
   ```
   : 교착상태가 발생할 가능성을 배제하지 않고 교착상태가 발생하면 적절히 피해나가는 방법
   ```

   - 발견 기법(Detection)
   ```
   : 시스템에 교착상태가 발생했는지 점검하여 교착상태에 있는 프로세스와 자원을 발견하는 것
   ```

   - 회복 기법(Recovery)
   ```
   : 교착상태를 일으킨 프로세스를 종료하거나 교착상태의 프로세스에 할당된 자원을 선점하여 프로세스나 자원을 회복하는 것
   ```

### Secure OS
```
: 보안 기능을 갖춘 커널을 이식하여 외부의 침입으로부터 시스템자원을 보호하는 운영체제
```
해당 OS의 보안 기능으로 식별 및 인증, 이므이적/강제적 접근통제, 객체 재사용 보호, 완전한 조정, 신뢰경로, 감사 및 감사기록 축소등이 있다.

- 보안커널
```
: 보안 기능을 갖춘 커널을 의미하며, TCB를 기반으로 참조 모니터의 개념을 구현하고 집행한다
```
   
## 2. 네트워크   
```
: 두 대 이상의 컴퓨터나 기기가 그물망처럼 연결되어 서로 데이터를 주고받을 수 있는 통신망
```
현대 컴퓨터의 발전을 이끌어준 부분이며, 운영체제에서도 중요하게 여겨지는 부분이다. 그리고 이 부분 덕에 보안에 대한 경각심과 중요도도 높아졌다. 2번째 챕터에서는 이 네트워크에 대해 알아보자

### OSI 참조 모델
```
: 다른 시스템 간의 원할한 통신을 위해 ISO (국제 표준화기구)에서 제안한 통신 규약을 가리킨다
```
7개의 계층으로 구분되며 아래와 같다  

#### 물리 계층
```
: 전송에 필요한 두 장치 간의 실제 접속과 절단 등 기계적, 전기적, 기능적, 절차적 특성에 대한 규칙을 정의한다
```

#### 데이터 링크 계층
```
: 두 개의 인접한 개발 시스템들간에 신뢰성 있고 효율적인 정보 전송을 할 수 있도록 시스템 간 연결 설정과 유지 및 종료를 담당한다
```

#### 네트워크 계층
```
: 개방 시스템들 가느이 네트워크 연결을 관리하는 기능과 데이터의 교환 및 중계 기능을 한다
```

#### 전송 계층
```
: 논리적 안정과 균일한 데이터 전송 서비스를 제공함으로써 종단 시스템 간에 투명한 데이터 전송을 가능하게 한다
```

#### 세션 계층
```
: 송수신 측 간의 관련성을 유지하고 대화 제어를 담당한다
```

#### 표현 계층
```
: 서로 다른 데이터 표현 형태를 갖는 시스템 간의 상호 접속을 위해 필요한 계층으로 코드변환, 데이터 암호화, 데이터 압축, 구문 섬색등의 기능을 수행한다
```

#### 응용 계층
```
: 사용자가 OSI 환경에 접근할 수 이도록 서비스를 제공한다
```

### 네트워크 관련 장비
네트워크의 발전은 소프트웨어가 아니라 하드웨어에서부터 시작된다. 하드웨어 목록을 살펴보자

#### 라우터
```
: 랜과 랜을 연결하고 데이터 전송의 최적 경로를 선택하는 기능을 가진 장치이다
```

#### 리피터
```
: 거리가 증가할수록 감쇠하는 디지털 신호의 장거리 전송을 위해 수신호 신호를 재생시키거나 출력 전압을 높여 전송하는 장치이다
```

#### 허브
```
: 가까운 거리의 컴퓨터들을 연결하는 장치로 각가그이 회선을 통합 관리하고 신호증폭 기능을 하는 장치이다
```

#### 브리지
```
: 랜과 랜을 연결하거나 랜 안에서의 컴퓨터 그룹을 연격하는 장치이다
```

#### 게이트웨이
```
: 전 계층의 프로토콜 구조가 다른 네트워크를 연결하고 다른 네트워크에 데이터를 보내거나 받아들이는 출입구 역할을 한다
```

### 네트워크 구성
#### IP주소
```
: 인터넷에 연결된 모든 컴퓨터 자원을 구분하기 위한 고유한 주소이다
```
- IPv4
8비트씩 4부분, 총 32비트로 구성됐으며, 네트워크 부분의 길이에 따라 A 클래스에서 E 클래스까지 총 5단계로 구성되어 있다.   

- IPv6
IPv4의 주소 부족 문제를 해결하기 위해 개발되었으며, 16비트씩 8부분 총 128비트로 구성됐으며, 각 부분을 16진수로 표현하고 콜론으로 구분한다. 인증성, 기밀성, 데이터 무결성의 지원으로 보안 문제를 해결할 수 있다.

#### 프로토콜
네트워크는 단순히 데이터를 송수신하지 않고 사전에 정해둔 규칙에 의거해 송수신하여야 한다. 그것을 **프로토콜**이라한다
```
: 서로 다른 기기들 간의 데이터 교환을 원화할게 수행할 수 있도록 표준화시켜 놓은 통신규약이다
```
프로토콜은 다음과 같은 기본요소를 가지고 있다

- 구문
```
: 전송하고자하는 데이터의 형식, 부호화, 신호레벨 등을 규정한다
```

- 의미
```
: 두 기기 간의 효율적이고 정확한 정보 전송을 위한 협조 사항과 오류 관리를 위한 제어 정보를 규정한다
```

- 시간
```
: 두 기기 간의 통신 속도, 메시지의 순서 제어등을 규정한다
```
   
#### 패킷 교환 방식
```
: 메시지를 일정한 길이의 패킷으로 잘라서 전송하는 방식이다
```

- 가상회선 방식
```
: 단말기 상호 간에 논리적인 가상 통신 회선을 미리 설정하여 송신자와 수신지 사이의 연결을 확립한 후에 설정된 경로를 따라 패킷들을 순서적으로 운반하는 방식이다
```
정보 전송 전에 제어 패킷에 의해 경로가 설정되며, 패킷의 송수신 순서가 같다   

- 데이터그램 방식
```
: 연결 경로를 설정하지 않고 인접한 노드들의 트래픽 상황을 감안하여 각각의 패킷들을 순서에 상관없이 독립적으로 운반하는 방식이다
```
패킷의 목적지의 완전한 주소를 가져야하며, 도착한 패킷들은 수신측에서 순서를 재정리한다   

#### 프로토콜 종류
1. TCP/IP
```
: 서로 다른기종의 컴퓨터들이 데이터를 주고받을 수 있도록 하는 표준 프로토콜이다
```
- TCP
전송계층에 해당하며, 가상 회선 방식을 기반으로하는 양방향 연결 서비스를 제공한다. 패킷의 **다중화, 순서제어, 오류제어, 흐름제어기능**을 제공한다

- IP
네트워크 계층에 해당하며 데이터 그램 방식을 기반으로하는 비연결형 서비스를 제공한다. 패킷의 **분해/조립, 주소지정, 경로 선택 기능**을 제공한다

2. UDP
```
: 데이터 전송 전에 연결을 설정하지 않는 비연결형 서비스를 제공하는 프로토콜이다
```

3. L2TP
```
: PPTP와 L2F의 기술적 장점들을 결합하여 만들어진 터널링 프로토콜이다
```

4. ICMP
```
: IP와 조합하여 통신중에 발생하는 오류의 처리와 전송 경로 변경 등을 위한 제어 메시지를 관리하는 역할을 한다
```

5. ARP/RARP
- ARP
```
: 호스트의 IP 주소를 호스트와 연결된 네트워크 접속 장치의 물리적 주소로 변환하는 기능을 수행하는 프로토콜
```

- RARP
```
:물리적 주소를 IP 주소로 변환하는 기능을 수행한다. 역순주소결정 프로토콜이라고도 한다
```
### 네트워크 구축
네트워크 구축 방법은 총 5가지 있다.

1. 성형
2. 링형
3. 버스형
4. 계층형
5. 망형

그리고 이 네트워크 구축에 사용되는 프로토콜들도 존재한다

1. NAT
```
: 한 개의 정식 IP 주소에 대량의 가상 사설 IP 주소를 할당 및 연결하는 기능이다
```
외부에 접속할 수 있는 노드는 어느 시멎에서 한 개로 제한되는 문제가 있지만 IP마스커레이드를 이용하여 해결할 수 있다

2. IGP
```
: 하나의 자율 시스템 내의 라우팅에 사용되는 프로토콜이다
```
- RIP
현재 가장 널리 사용되는 라우팅 프로토콜로, 벡터 라우팅 프로토콜이라고도 한다
최단 경로 탐색에 Bellman-Ford 알고리즘을 사용하고, 가장 효율적인 방법이다

- OSPF
RIP의 단점을 해결하여 새로운 기능을 지원하는 인터넷 프로토콜로 대규모 네트워크에서 사용된다   
최단 경로 탐색에 다익스트라 알고리즘을 사용한다

3. EGP
```
: 자율 시스템 간의 라우팅에 사용되는 프로토콜이다
```
4. BGP
```
: EGP의 단점을 보완하기 위해 만들어진 프로토콜이다
```
초기에 연결될 때에는 전체경로 제어표를 교환하고, 이후에는 변화된 정보만을 교환한다

### 네트워크 제어

#### 흐름제어
```
: 네트워크 내의 원활한 흐름을 위해 송,수신 측 사이에 전소오디는 패킷의 양이나 속도를 규제하는 기능이다
```

- 정지대기
```
: 수신 측의 확인 신호를 받은 후에 다음 패킷을 전송하는 방식이다
```

- 슬라이딩 윈도우
```
: 확인 신호, 즉 수신 통지를 이용하여 송신 데이터의 양을 조절하는 방식이다.
```
