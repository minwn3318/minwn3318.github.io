# 정보처리기사 - 데이터 입출력 구현

해당 포스트에서는 정보처리기사 개념문제집 2장 **데이터 입출력 구현**을 정리한다

## 시스템 카탈로그
```
: 시스템 그 자체에 관련이 있는 다양한 객체 관한 정보를 포함하는 시스템 데이터베이스이다
```
- 시스템 카탈로그 내의 가 테이블은 사용자를 포함하여 DBMS에서 지원하는 모든데이터 객체 대한 정의나 명세 관한정보를 유지 관리하는 시스템 테이블이다
- 타칼로그들이 생성되면 데이터 사전에 저장되ㅣ 때문에 좁은 의미로는 카탈로그를 데이터 사전이라고도 한다

### 트랜잭션의 특성
- 원자성
> 트랜잭션의 연산으 데이터베이스에 반영되도록 완료되즌디 아니면 전혀 반영되지 않도록 복구되어야 함

- 일관성
> 트랜잭션이 그 실행을 성공적으로 완료하면 언제나 일관성 있는 데이터베이스 상태로 변환함

- 독립성
> 둘 이상의 트랜잭셩이 동시에 병해 실행되는 경우 어느 하나의 트랜잭션 시행중에 다른 트랜잭션의 연산이 끼어들 수 없음

- 영속성
> 성공적으로 완료된 트랜잭션의 결과는 시스템이고장나더라도 영구적으로 반영되어야 함

---

## CRUD 분석
```
: 프로세스와 테이블 간에 CRUD 매트릭슬르 만들어서 트랜잭션을 분석하는것
```
- 많은 트랜잭션이 몰리는 테이블을 파악할 수있으르모 디스크 구성 시 유용한 자료로 활용할 수 있다
- CRUD 매트릭스의 각 셀에는 생성 읽기 갱싱 삭제의 앞 글자가 들어간다

### 인덱스
```
: 데이터 레코드를 빠르게 접근하기 위해 <키 값, 포인터> 쌍으로 구성되는 데이터 구조이다
```
- 인덱스는 레코드가 저장된 물리적 구조에 접근하는 방법을 제공한다
- 인덱스를 통해서 파일의 레코드에 빠르게 액세스 할 수 있다

## 뷰
```
: 사용자에게 접근이 허용된 자료만을 제한적으로 보여주기 위해 하나 이상의 기본 테입르로부터 유도된 이름을 가지는 가상 테이블이다
```
- 뷰가 정의된 기본 테이블이나 뷰를 삭제하면 그 테입르이나 뷰를 기초로 정의된 다른 뷰도 자동으로 삭제된다
- 뷰를 정의할 때는 CREATE문, 제거할 때는 DROP문을 사용한다

### 파디션의 종류
- 범위 분할
> 지정한 열의 값을 기준으로 분할힘

- 해시 분할
> 해시 함수를 적용한 결과 값에 띠라 데이터를 분할 함
> - 특정 파티션에 데이터가 집중되는 범위 분할의 단점을 보완한 것으로, 데이터를 고르게 분산할 때 유용함
> - 특정 데이터가 어디에 있는지 판단할 수 없음
> - 고객번호, 주민번호 등과 같이 데이터가 고른 컬럼에 효과적임

- 조합 분할
> 범위 분할로 분할한 다음 해시 함수를 적용하여 다시 분할하는 방식
> - 범위 분할한 파티션이 너무 커서 관리가 어려울 때 유용함

## 분산 데이터베이스의 목표
- 위치 투명성
> 액세스하려는 데이터베이스의 실제 위치를 알 필요 없이 단지 데이터베잇의 논리적인 명칭만으로 액세할 수 있음
- 중복 투명성
> 동일 데이터가 여러 곳에 중복되어 있더라도 사용자는 마치 하나의 데이터만 존재하는 것처럼 사용하고 시스템은 자동으로 여러 자료에 대한 작업을 수행함
- 병행 투명성
> 분산 데이터베이스와 관련된 다수의 트랜잭션들이 동시에 실현되도라도 그 트랜잭션의 결과는 영향을 받지 않음
- 장애투명성
> 트랜잭션, DBMS, 네트워크, 컴푸터 장애에도 불구하고 트랝개션을 정화하게 처리함

## RTO/RPO
- RTO
> 비상사태 또는 업무 중단 시점으로부터 복구되어 가동될 때까지의 소요 시간을 의미함
- RPO
> 비상사태 또는 업무 중단 시점으로부터 데이터를 복구할 수 있는 기준점을 의미함

## 임의 접근통제
```
데이터에 접근하는 사용자의 신원에 따라 접근 권한을 부여하는 방식
```
- 데이터 소유자가 접근통제 권한을 지정하고 제어한다
- 객체를 생성한 사용자가 생성된 개체 대한 모든 권한을 부여받고, 부여된 권한을 다른 사용자에게 허가할수도 있다

### 강제 접근통제
```
: 주체와 객체의 등급을 비교하여 접근 권한을 부여하는 방식
```
- 시스템이 접근통제 권한을 지정한다
- 데이터베이스 객체별로 보완 등급을 부여할 수 있다
- 사용자별로 인가 등급을 부여할 수 있다

### 역할기반 접근통제
```
: 사용자의 역할에 따라 접근 권한을 부여하는 방식
```
- 중앙관리자가 접근통제 권한을 지정한다
- 임의 접근토엦와 강제 접근통제의 단점으 보완하였다
- 다중 프로그래밍 환경에 최적화된 방식이다

---

## DAS
```
: 서버와 저장장치를 전용 케이블로 직접 연결하는 방식
```
- 외장하드를 연결하는 것이 해당됨
- 직접 연결 방식이므로 다른 서버에서 접근할 수 없고 파일을 공유할 수 없다

### SAN
```
: 서버와 저장장치를 연결하는 전용네트워크를 별도로 구성하는 방식
```
- 파이버 채널 스위치를 이요하여 네트워크를 구서한다
- 처리속도가 빠르다
- 서버들이 저장장치 및 파일을 공유할 수 있다
