# 정보처리기사 - 소프트웨어 개발 보안 구축   

해당 포스트에서는 정보처리기사 개념문제집의 9장 **소프트웨어 개발 보안 구축 암호편**을 정리한다   

## 5. 암호 알고리즘   
```
: 중요 정보를 보호하기 위해 평문을 암호화된 문장으로 만드는 절차 또는 방법을 의미한다
```   
단순히 비밀유지를 위해 사용하기도 하지만, 정보화 사회에서는 **보안의 3대 요소**를 지키기 위해 적극적으로 사용된다.   
   
암호 방식 분류로는 다음과 같다   
   
- 암호화 방식

- - 양방향
- - - 개인키
- - - - 스트림 방식
- - - - 블록 방식

- - - 공개키

- - 단반향
- - - 해시

이어서 각 암호화 방식에 대해 설명하도록 한다.   

### 개인키 암호화 기법   
```
: 동일한 키로 데이터를 암호화하고 복호화하는 암호화 기법을 의미한다
```   
개인키 암호화 기법은 **대칭 암호 기법** 또는 **단일키 암호화 기법**이라고도 한다. 암호화/복호화 속도가 빠르다는 장점이 있으나, 사용되는 대상이 많아질수록 관리해야 할 키의 수가 많아진다는 단점이 존재한다.   
  
방식에는 **스트림 암호화 방식**과 **블록 암호화 방식**이 존재한다

- 스트림 암호화 방식
```
: 평문과 동일한 길이의 스트림을 생성하여 비트 단위로 암호화하는 방식을 가리킨다
```
종류로 LFSR, RC4, TKIP가 존재한다.   

- 블록 암호화 방식
```
: 한 번에 하나의 데이터 블록을 암호화 하는 방식을 가리킨다
```
종류로 DES, SEED, AES, ARIA, IDEA, Skipjack

### 공개키 암호화 기법   
```
: 암호화 할 때 사용하는 공개키를 사용자에게 공개하고, 복호화 할 때의 비밀키를 관리자가 비밀리에 관리하는 암호화 기법을 의미한다
```
공개키 암호화 기법은 **비대칭 암호 기법**이라고도 한다. 사용되는 대상이 많아져도 관리해야할 키의 수가 적다즌 장점이 있으나, 암호화/복호화 속도가 느리다는 단점이 존재한다   
대표적으로 RSA와 ECC기법이 있다.

### 알고리즘의 종류   

- IDEA
```
: 스위스의 라이와 메시가 1990년에 개발한 PES를 개선한 알고리즘이다
```
블록 크기는 64비트이고, 키 길이는 128비트이다.

- SkipJack
```
: 국가 안전 보장국 (NSA)에서 개발한 암호화 알고리즘이다
```
**클리퍼 칩**이라는 IC 칩에 내장되어 있다.
블록 크기는 64비트이고, 키 길이는 80비트이다.
주로 **음성 통신 장비**에 삽입되어, 음성 데이터를 암호화한다.

- DES
```
: 1975년 미국 NBS에서 발표한 개인키 암호화 알고리즘이다
```
블록크기는 64비트, 키 길이는 56비트이며 16회의 라운드를 수행한다

- AES
```
: 2001년 미국 표준 기술 연구소 (NIST)에서 발표한 개인키 암호화 알고리즘이다
```
DES의 한계를 느낀 NIST에서 공모한 후 발표하였다.
블록 크기는 128비트이며, 키 길이에 따라 AES-128, AES-192, AES-256으로 분류된다

- RSA
```
: 1978년 MIT의 라이베스트, 샤미르, 애들먼에 의해 제안된 공개키 암호화 알고리즘이다
```
큰 숫자를 소인수분해 하기 어렵다는 것에 기반하여 만들어졌다.

- TKIP
```
: 기존의 무선 랜 보안 프로토콜인 WEP의 취약성을 보완한 데이터 보안 프로토콜이다
```
암호 알고리즘의 입력 키 길이를 128비트로 늘리고 패킷당 키 할당, 키값 재설정 등의 키 관리 방식을 개선하였다.

### 해시   
```
: 입력 데이터나 메시지를 고정된 길이의 값이나 키로 변환하는 것을 의미한다
```
**해시 알고리즘**을 **해시 함수**라고 부르며, 해시 함수로 변환된 값이나 키를 **해시값** 또는 **해시키**라고 부른다.
종류로 SHA 시리즈, HAVAL, MD4, MD5, N-NASH, SNEFRU가 있다

- MD4
```
: 1991년 R Rivest가 MD4를 대처하기 위해 고안한 암호화 해시 함수이다
```
블록크기가 512비트이며, 키 길이는 128비트이다.
